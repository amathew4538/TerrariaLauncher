# Contributing Guidelines
Just a few good gudelines to follow.
## Creating a good pull request
Make sure your pull request is:
- Easy to understand
- Provides enough information about:
  - Your edits (a summary is fine)
  - What you are fixing or adding
  - What platforms it is confirmed to work on
  - Any other important info
- Includes necessary pictures of the edit
- Is completely safe
